Signal Translate Bot

Deploy a privacy-preserving auto-translation bot for Signal groups, running inside an Intel TDX Trusted Execution Environment.
TEE-Protected LibreTranslate Signal CLI Phala Cloud
Overview

Architecture

+------------------------------------------------------------------+ | TEE Boundary (Intel TDX) | | | | Signal CLI Signal Bot LibreTranslate | | (decrypts E2E) --> (detects lang) --> (translates) | | | | | | | | v | | | | quoted reply | | | | with flag emoji | | | | | | | | All plaintext stays inside encrypted memory | +------------------------------------------------------------------+
Privacy Guarantee
Messages are decrypted, translated, and re-encrypted entirely within TEE-protected memory. The operator never sees plaintext.
4 Containers
signal-api, signal-bot, libretranslate, and registration-proxy all run inside the same TEE enclave.
Step 0

Prerequisites

  1. 01 Docker and Docker Compose installed
  2. 02 A phone number for Signal registration prepaid SIM recommended
  3. 03 NEAR AI API key from https://near.ai
  4. 04 Phala Cloud account + API token from https://cloud.phala.network
  5. 05 DockerHub account (to push the bot image)
  6. 06 Phala CLI installed
npm install -g phala
phala auth login YOUR_API_TOKEN
The API token comes from: Phala dashboard > P logo > API Tokens > Create Token. This is NOT the same as an sk-rp-... registry key.
Step 1

Build & Push Images

Phala CVMs run linux/amd64. If you're on an ARM Mac (M1/M2/M3/M4), you MUST cross-compile.
# Build signal-bot for linux/amd64
docker buildx build --platform linux/amd64 \
  -t YOUR_DOCKERHUB/signal-bot-tee:latest \
  -f docker/Dockerfile --push .

# Build registration proxy for linux/amd64
docker buildx build --platform linux/amd64 \
  -t YOUR_DOCKERHUB/signal-registration-proxy:latest \
  -f docker/Dockerfile.proxy --push .
Other images are pulled from public registries
bbernhard/signal-cli-rest-api (pinned by SHA256 digest)
libretranslate/libretranslate:v1.6.3
Step 2

Configure phala-compose.yaml

Update the image references to point to your DockerHub:

# docker/phala-compose.yaml

signal-bot:
  image: YOUR_DOCKERHUB/signal-bot-tee:latest
  environment:
    # Translation settings
    - TRANSLATION__ENABLED=true
    - TRANSLATION__LIBRETRANSLATE_URL=http://libretranslate:5000
    - TRANSLATION__GROUPS=${TRANSLATION_GROUPS}
    - TRANSLATION__ALSO_CHAT=false

libretranslate:
  image: libretranslate/libretranslate:v1.6.3
  environment:
    - LT_LOAD_ONLY=${LT_LOAD_ONLY:-en,es}
LibreTranslate runs inside the TEE boundary with no external ports. Plaintext translations never leave encrypted memory.
Step 3

Deploy to Phala Cloud

cd docker

phala cvms create \
  --name signal-translate-bot \
  --compose ./phala-compose.yaml \
  --vcpu 2 \
  --memory 4096 \
  --disk-size 20

When prompted, enter your encrypted secrets:

VariableValue
NEAR_AI_API_KEYYour NEAR AI API key
TRANSLATION_ENABLEDtrue
TRANSLATION_GROUPSGROUP_ID:en:es set after registration
LT_LOAD_ONLYen,es (or your language pair)
Upgrading an existing CVM?
phala deploy --uuid YOUR_CVM_UUID --compose ./phala-compose.yaml
Step 4

Register Signal Phone Number

Get your CVM endpoint first:

phala cvms get APP_ID
# endpoint: https://[app-id]-8080.dstack-prod5.phala.network

Get a captcha from signalcaptchas.org/registration/generate.html, then:

# Request verification code
curl -X POST https://[endpoint]-8080/v1/register/+1YOURNUMBER \
  -H "Content-Type: application/json" \
  -d '{"captcha": "signalcaptcha://..."}'

# Enter the SMS code
curl -X POST https://[endpoint]-8080/v1/register/+1YOURNUMBER/verify/CODE

# Set profile name
curl -X PUT https://[endpoint]-8080/v1/profiles/+1YOURNUMBER \
  -H "Content-Type: application/json" \
  -d '{"name": "Translate Bot"}'
Step 5

Get Your Group ID

Add the bot's phone number to your Signal group, then find the group ID:

# Option A: List groups via Signal CLI API
curl https://[endpoint]-8080/v1/groups/+1BOTNUMBER

# Option B: Send a message in the group, then check logs
phala cvms logs APP_ID
# Look for: group_id = "abc123..."

Then update the TRANSLATION_GROUPS secret:

# Format: GROUP_ID:LANG_A:LANG_B
# Multiple groups: GROUP1:en:es,GROUP2:en:fr

TRANSLATION_GROUPS="dGhpcyBpcyBhIGdyb3VwIGlk:en:es"
Redeploy with the group ID
phala deploy --uuid APP_ID --compose ./phala-compose.yaml

Set TRANSLATION_GROUPS in the encrypted secrets when prompted.
Step 6

Choose Language Pairs

Set LT_LOAD_ONLY to download only the models you need (saves RAM and startup time):

Common Pairs
en,es - English / Spanish
en,fr - English / French
en,de - English / German
en,pt - English / Portuguese
en,zh - English / Chinese
en,ar - English / Arabic
en,ja - English / Japanese
en,ko - English / Korean
Multi-Group Example
TRANSLATION_GROUPS=
group1:en:es,
group2:en:fr,
group3:es:pt

LT_LOAD_ONLY=en,es,fr,pt
Each language model is ~50-100MB. Loading all languages needs ~2GB+ RAM. Use LT_LOAD_ONLY to keep memory low.
Step 7

Verify & Test

  1. 01 Send an English message in the configured group
  2. 02 Bot replies with a quoted Spanish translation + flag emoji
  3. 03 Send a Spanish message - bot replies with English translation
  4. 04 Send !verify test123 to get TEE attestation proof
  5. 05 Commands like !help still work in translation groups
You: "Hello, how is everyone doing today?" Bot: replying to your message 🇪🇸 Hola, como estan todos hoy?
If also_chat is false (default), the bot ONLY translates in configured groups. No AI chat responses. Set TRANSLATION__ALSO_CHAT=true to enable both.
Step 8

Verify TEE Attestation

Prove that LibreTranslate + Signal CLI + Bot are all inside the TEE:

  1. 01 Send !verify my-nonce-123 to the bot
  2. 02 Bot returns App ID, compose hash, and TDX quote
  3. 03 Visit https://proof.t16z.com and enter the App ID
  4. 04 Confirm compose hash matches your phala-compose.yaml
  5. 05 The compose hash covers ALL containers - including LibreTranslate

Troubleshooting

IssueSolution
No translationCheck TRANSLATION__ENABLED=true and group ID is correct
LibreTranslate slow startFirst boot downloads models (~2 min). Check LT_LOAD_ONLY.
Signal reg failsCaptcha expires fast. Get a fresh one immediately before POST.
Bot translates itselfShould not happen (self-message filter). Check logs for source == receiving_account.
Short msgs ignoredMessages under 3 chars are skipped. This is by design.
Wrong language detectedVery short or mixed-language text has low confidence. Minimum threshold is 0.3.
OOM / high memorySet LT_LOAD_ONLY to only your needed languages. Consider --memory 8192.
# Check logs
phala cvms logs APP_ID

# Test LibreTranslate directly
curl -X POST http://localhost:5000/translate \
  -H "Content-Type: application/json" \
  -d '{"q":"Hello","source":"en","target":"es"}'
